Client and Server, Python

For this project, I wanted to build upon what I am learning in my computer networks course so I looked to build programs that mimic a client and a server. This projects involved an understanding of python, http requests, sockets, caches, web proxies, and client-server relationships.

My Approach & How It Works

There are two programs echo_client and echo_server. The client program forwards http get requests to the server, which is acting as a web proxy. The server receives the request. It checks if the url in the request is already in its cache. If it is, it will forward the request with an additonal 'If-Modified-Since:' line. Based on the response it gets back from the web, it will either send back to the client a new response or the response it has stored in its cache. If the url is not in the cache, the server will forward the request unmodified. It will send back to the client the response it receives from the web (we expect that that is a 200 OK message with an html message of the website)

How to Use My Code

Download echo_server and echo_client. Open two terminal windows. Travel to the location containing the application (typically /Downloads). In one window, run the command: python3 echo_server.py This will open the server. In the next window run: pyhton3 echo_client.py You will be prompted for the host and path of the website you want to visit.

How to use your browser as a client

Open up a browser. I reccomend using Chrome. Change your proxy settings so that the broswer is listenig to 'localhost':50008. Open a terminal window and run echo_server. Type in an address in your browser.

Next Steps

These programs encodes and decodes messages using 'utf-8' so not all webpages will load because a Unicode decode error will occur. In addition, these programs only process http get request so a next step would be further building the programs to further other http requests.
